# O que eu aprendi de git:

- Serve para versionar o codigo para melhor organização
- Pode ou não ter varias brunchs(galhos) para poder organizar melhor o trabalho em equipe e manter o codigo principal mais seguro, mas para projetos pequenos, pessoais e etc não é tão necessario (inclusive nessas primeiras versões nao serão usadas)>
- Algumas funções basicas que eu aprendi:

git init: Inicia um novo repositorio git.
git add .: Adiciona todos os arquivos ao proximo commit.
git commit -m "mensagem": Versiona aqueles arquivos com a mensagem escrita.
git remote add origin URL: Conecta ao local remoto da URL.
git pull: Atualiza os arquivos locais com os arquivos mais atuais do local remoto conectado.
git push: Sobe os arquivos para o local remoto na branch em que você está.
